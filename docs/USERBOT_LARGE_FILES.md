# Userbot-скрипт для загрузки больших файлов (>50 МБ) на Яндекс.Диск

## Зачем это нужно

Официальный **Telegram Bot API** (через который работает этот бот) ограничивает скачивание файлов через `getFile`: максимум **около 50 МБ**. Файлы больше этого размера Telegram возвращает с ошибкой `Bad Request: file is too big`, и бот не может их забрать.

**Userbot** — это клиент Telegram, который подключается как **обычный пользователь** (по протоколу MTProto), а не как бот. У пользовательского API лимит на скачивание файлов **до 2 ГБ**. Поэтому отдельный скрипт‑userbot может:

- получать сообщения/файлы из диалогов или групп (куда добавлен этот пользователь);
- скачивать большие файлы;
- загружать их на Яндекс.Диск (или передавать путь/файл основному боту).

Так можно обрабатывать файлы 50 МБ–2 ГБ без поднятия своего Bot API сервера.

---

## Как это устроено

1. **Отдельный аккаунт Telegram**  
   Для userbot нужен свой номер телефона (или виртуальный). Этот аккаунт добавляют в нужные группы/чаты. Бот и userbot могут работать параллельно: бот обрабатывает файлы до 50 МБ, userbot — большие.

2. **Библиотеки**  
   Используются клиенты Telegram для пользователей (не Bot API):
   - **Telethon** (https://github.com/LonamiWebs/Telethon) — асинхронный, удобен с asyncio;
   - **Pyrogram** (https://github.com/pyrogram/pyrogram) — тоже асинхронный, современный API.

3. **Сценарии работы**
   - **Вариант A:** Userbot слушает те же группы, что и бот. Если приходит файл больше 50 МБ, бот его пропускает (как сейчас), а userbot перехватывает сообщение, скачивает файл и заливает на Яндекс.Диск (нужен общий конфиг: токен Я.Диска, папка и т.д.).
   - **Вариант B:** Пользователь пересылает большие файлы в отдельный чат/канал с userbot. Userbot скачивает и загружает на Диск, при необходимости уведомляет пользователя (например, через бота в ЛС).

4. **Безопасность и риски**
   - Аккаунт userbot подчиняется правилам Telegram для пользователей. За спам, массовые рассылки или подозрительную активность аккаунт могут ограничить или заблокировать.
   - Нужно хранить **api_id/api_hash** (получают на my.telegram.org) и сессию (логин/пароль или session-файл) в безопасном месте, не светить их в репозиторий.
   - Обычно userbot запускают на том же сервере, что и бота, с общим доступом к конфигу (токен Я.Диска, путь к папке). Передавать файлы между процессами можно через общую папку или очередь (Redis/БД), либо userbot сам дергает API Яндекса.

## Минимальный пример (Telethon + загрузка на Яндекс.Диск)

Установка:

```bash
pip install telethon aiohttp
```

Скрипт (псевдокод идеи):

```python

from telethon.events import NewMessage
import aiohttp
import os

API_ID = 12345  # с my.telegram.org
API_HASH = "..."

client = TelegramClient(SESSION_NAME, API_ID, API_HASH)

async def upload_to_yandex_disk(file_path, disk_path):
    # Реализация через REST API Яндекса (как в yandex_disk.py)
    # Получить upload URL, загрузить файл по частям и т.д.
    pass

@client.on(NewMessage(incoming=True, func=lambda e: e.document and e.document.size > 50 * 1024 * 1024))
async def on_large_file(event):
    # Событие: новое входящее сообщение с документом > 50 МБ
    doc = event.document
    local_path = await event.download_media()
    disk_path = f"{ROOT_FOLDER}/{doc.date:%Y/%m}/{doc.attributes[0].file_name}"
    await upload_to_yandex_disk(local_path, disk_path)
    os.remove(local_path)
    # При желании отправить уведомление пользователю (через бота или в чат)
    await event.reply("Файл загружен на Яндекс.Диск.")

client.start()  # при первом запуске запросит номер и код
client.run_until_disconnected()
```

Детали (получение `upload_url`, разбиение на чанки, обработка ошибок) делаются по документации Яндекс.Диск API и вашему существующему коду в `yandex_disk.py`.

## Итог

- **Userbot** даёт обход лимита 50 МБ за счёт работы от имени пользователя (до 2 ГБ).
- Нужен отдельный аккаунт Telegram, Telethon или Pyrogram, и логика загрузки на Яндекс.Диск (можно переиспользовать ваш проект).
- Риски: правила Telegram для пользователей, хранение api_id/api_hash и сессии в безопасности.
- Для текущего бота ничего менять не обязательно: большие файлы по-прежнему пропускаются без постановки в очередь; userbot — опциональное дополнение для тех, кому нужны именно большие файлы.

Если решите внедрить такой скрипт, можно вынести общую логику (путь на диске, токен, загрузка) в отдельный модуль и вызывать его и из бота, и из userbot.
